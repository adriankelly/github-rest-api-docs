# Authentication

Authentication confirms who is making the request.

For the GitHub REST API, endpoints work differently based on if the request is unauthenticated or authenticated.

This section explains:

- when authentication is required
- the difference between authenticated and unauthenticated requests
- how bearer tokens are used
- common authentication-related errors such as `401 Unauthorized` and `403 Forbidden`
